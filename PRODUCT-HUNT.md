## Description

NotebookLM is one of the most powerful research tools available — but it lives in a browser tab, disconnected from your actual work.

notebooklm-skill bridges that gap. Install it once, and your AI coding assistant (Claude Code, Cursor, or OpenAI Codex) can:

— Create a NotebookLM notebook for each project, automatically
— Sync your docs, READMEs, case studies, and live URLs as sources
— Answer questions about your codebase backed by your actual documentation
— Generate podcasts, reports, quizzes, flashcards, slide decks, and mind maps from your project knowledge
— Save end-of-session summaries to the project's notebook — building a searchable history over time

The core idea: every project gets its own notebook, linked via a `.notebooklm/config.json` file. When you run `/notebooklm` inside a project, your AI automatically uses the right notebook — no IDs to remember, no context to re-explain.

Open source. MIT. Built on notebooklm-py by @teng-lin.

## First comment (pin as maker)

Hey Product Hunt 👋

I built this because I kept switching between my AI coding assistant and NotebookLM — both powerful, completely disconnected.

The fix was obvious once I saw it: every project should have its own notebook, and the AI should know which one to use automatically.

What you get:
→ `/notebooklm init` — creates a notebook, scans the project for docs, adds them as sources
→ `/notebooklm` — ask anything, generate artifacts (podcast, report, quiz, slides...), sync sources
→ `/wrapup` — end-of-session log pushed to the project's notebook, building searchable history over time

Works natively with Claude Code, with adapters for Cursor and OpenAI Codex (both use the same `/skill-name` trigger system — turns out they're more compatible than I expected).

Would love to know what AI assistants you want adapters for next.
